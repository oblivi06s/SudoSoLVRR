About SudoPHASE

Ant colony optimization (ACS / multi-colony / parallel) · Sudoku solver and player

Rules of Sudoku

Sudoku is a number-placement puzzle. The goal is to fill the grid so that:

  • Every row contains each digit (1–9 for 9×9, 1–16 for 16×16, 1–25 for 25×25) exactly once.
  • Every column contains each digit exactly once.
  • Every sub-grid (block) contains each digit exactly once. In 9×9 the blocks are 3×3; in 16×16 they are 4×4; in 25×25 they are 5×5.

Some cells are pre-filled (clues). You fill the empty cells without changing the clues. A valid puzzle has exactly one solution.

How to use the application

Main menu

  • Play — Open the puzzle screen. Choose a puzzle from the library, use one you created, or play an uploaded instance, then solve it yourself or use the solver.
  • Create Puzzle — Build your own puzzle: pick a size (9×9, 16×16, or 25×25), fill in the clues, then click “Play this puzzle” to open it in Play mode.
  • Upload Puzzle — Select a text file in the same format as the library instances; the puzzle is loaded directly into the Play screen.
  • About Game — This page: rules and usage.

Play screen

Puzzle library (left panel)

  • Select Size (9×9, 16×16, or 25×25). The list shows files from instances/paquita-database (9×9), instances/16x16-database (16×16), or instances/25x25 (25×25).
  • Click an instance in the list, then Load selected to load it into the grid.

Parameters

  • Algorithm — Choose 0 (single-thread ACS), 3 (multi-colony), or 4 (multi-thread multi-colony). Thread and communication settings apply mainly to algorithm 4.
  • Timeout (s) — Maximum time (seconds) the solver may run before stopping.
  • Ant colony — q0 (Greediness, --q0), ρ (Pheromone Persistence, --rho), ξ (Local Pheromone Update, --xi), and ρBVE (Best Value Evaporation, --evap).
  • Multi-Colony ACO — numacs, entropythreshold, and convthreshold (multi-colony–related settings in the UI).
  • Threads & comm — Shown for algorithm 4: parallel thread count and communication intervals passed to the C++ executable.
  • SOLVE PUZZLE — Runs the built-in solver. For algorithm 4 you may see live best-so-far cell counts from the solver’s progress output; the filled grid updates when a complete solution is found (digits may animate in). If the timeout is reached first, the last parsed partial or final state is shown when available.

Grid and input

  • Click a cell to select it (highlighted border). Pre-filled clues are fixed and cannot be edited or erased.
  • When you select a non-fixed cell, all cells in the same row, column, and block are highlighted as peers (with light blue shading in the light theme).
  • Digit buttons — Click a digit (1–9, 1–16, or 1–25) to place it in the selected cell. Only empty or non-fixed cells accept input.
  • Undo — Reverts the last change (digit entry, erase, load, or solve).
  • Erase — Clears the selected cell if it is not a fixed clue.
  • If a digit you place breaks the rules (duplicate in row, column, or block), that digit and any duplicates are highlighted in red until you fix them.

Clear — Empties all cells (under the grid). Use this to start over on the same size; fixed clues are removed.

Create Puzzle screen

  • Choose a size (9×9, 16×16, or 25×25).
  • Fill in the clue cells using the digit buttons. Empty cells will be the ones the player must solve.
  • Click Play this puzzle to open the puzzle in Play mode so you can solve it or run the solver.

Tips

  • Use the puzzle library to try different instances. Increase timeout or threads if the solver does not finish in time.
  • When creating a puzzle, make sure clues do not repeat in the same row, column, or block; otherwise the puzzle has no valid solution.
  • Use the theme toggle in the top-right corner (Light / Dark) to switch between light and dark modes; your choice is remembered across pages.
